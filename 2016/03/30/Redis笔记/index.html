Redis 笔记

文档 Redis 命令参考 非常详细，学习的内容主要源自于文档。

基础总结篇

数据结构的操作和特点

Key

Redis 里所有单个命令都属于原子操作。

  • 查询:
    • KEYS partten, 可使用通配符?, *, []，时间复杂度 O(n)。 在 key 很多的情况下，每次查询都会很慢，踩过这个坑，慎用。
  • 序列化和反序列化
    • DUMP key，得到序列化的值，和RDB格式一致
    • RESTORE key ttl dumped-value [replace]，反序列化。dumped-value 是序列化的值，ttl 设置成 0 表示不超时，replace 在 3.0 之后的版本出现，表示替换已存在的 key
  • 迁移
    • MIGRATE host port key destination-db timeout [COPY] [REPLACE]，跨实例的迁移数据，主要用于主从备份
    • MOVE key db，在当前实例迁移数据

  • 超时

    • EXPIRE key seconds，让 key 在指定时间之后被删除，时间单位是秒，如果是P开头，如PEXPIRE，时间单位是毫秒。
    • EXPIREAT KEY timestamp，在指定的时间被删除，timestamp 表示Unix时间戳。
    • TTL key，查看超时剩余时间。-2 表示该 key 不存在
    • PERSIST key，移除给定 key 的生存时间。

  • 排序

    • 特性
      • 使用 SORT 命令，返回或保存给定列表、集合、有序集合 key 中经过排序的元素。
      • 默认以数字进行排序, 可指定成字符
      • 支持 LIMIT，支持外键排序

  • 遍历迭代

String

  • 位操作
    • SETBIT key offset val: 设置 key 指定二进制偏移量的位
    • GETBIT key offset: 获取 key 指定二进制偏移量的位
    • BITCOUNT key: 统计 key 二进制位不为 0 的数目
    • BITOP operation destkey key [key ...]: 对 key 进行位操作
      — BITOP AND destkey key [key ...] ，对一个或多个 key 求逻辑并，并将结果保存到 destkey
      • BITOP OR destkey key [key ...]
      • BITOP XOR destkey key [key ...]
      • BITOP NOT destkey key
  • 字符串操作
    • STRLEN key: 获取字符串长度
    • MSET k1 v1 k2 v2: 设置多个 k, v 键值对
    • MGET k1, k2: 获取多个 key 对应的 value
    • APPEND key value: key 对应的 value 添加字符串，如果 key 不存在会新建 key
  • 数值操作
    • INCR key: value_of_key += 1
    • INCRBY key increment: value_of_key += int(increment)
    • INCRBYFLOAT key increment: value_of_key += float(increment)
    • DECR key: value_of_key -= 1
    • DECRBY key increment: value_of_key -= int(increment)

Hash

  • HSET hash key val: 名为hash的哈希表设置(k, v)键值对
  • HGET hash key: 返回 key 对应的 value
  • HGETALL hash: 获取整张表
  • HMSET hash key1 val1 key2 val2:
  • HMGET hash key1 key2:
  • HDEL hash key: 删除 key
  • HKEYS hash: 所有 keys
  • HVALUES hash: 所有 values
  • HLEN hash: hash 表长度
  • HSETNX hash key val: key 不存在的时候设置 value，如果已存在返回 nil
  • HEXISTS hash key: key 是否存在于 hash 表中

List

  • LPUSH list: 插入到 list 列表头
  • LPUSHX list item: 仅当列表 list 存在时执行LPUSH操作
  • LPOP list: list 列表头移出
  • RPUSH list: 插入到 list 列表尾
  • RPUSHX list: 仅当列表 list 存在时执行RPUSH操作
  • RPOP list: list 列表尾移出
  • RPOPLPUSH source dest: 将两个操作并为一个原子操作。相当于 rpop source|lpush dest
  • BRPOPLPUSH source dest timeout: RPOPLPUSH 的阻塞版本，超时返回 nil
  • LLEN list: len(list)
  • LRANGE list start end: list[start:end]
  • LINDEX list index: list[index]
  • LSET list index value: list[index] = value
  • LTRIM list start end: list = list[start:end+1]
  • LINSERT list BEFORE|AFTER pivot value: pivot 存在于 list 列表中时，把 value 插入到 pivot 之前或之后

Set

  • SADD set item1 [item2 ...]: 添加元素
  • SPOP set: 随机获取元素但不删除
  • SMOVE set1 set2 item: 将 item 从 set1 移动到 set2
  • SREM set item1 [item2 ...]: 移除元素
  • SMEMBERS set: 返回集合中所有元素
  • SISMEMBER set key: 判断 key 是否是 set 集合中的元素
  • SDIFF set1 set2 [set3 ...]: 返回给定集合的差集
  • SINTER set1 set2 [set3 ...]: 返回给定集合的交集
  • SUNION set1 set2 [set3 ...]: 返回给定集合的并集
  • SCARD set: 返回 set 长度
  • SDIFFSTORE dest set1 set2: 差集结果存储在 dest 中
  • SINTERSTORE dest set1 set2: -
  • SUNIONSTORE dest set1 set2: -
  • SRANDMEMBER dest set1 set2: -

SortedSet

  • ZADD key score1 item1[score2 item2 ...]: 将 score 和 item 加入到 key 的有序集中
  • ZSCORE key member: 返回有序集 key 中，成员 member 的 score 值。
  • ZRANGE key start end [withsocre]: 返回 key 中序号从 start 到 end 范围内的元素，指定了 withsocre 返回 score。
  • ZREVRANGE key start stop [WITHSCORES]: 返回从大到小排序后结果
  • ZRANGEBYSCORE key min max [WITHSCORES] [LIMIT offset count]: 返回 score 在 min 到 max 中的所有元素，可 limit
  • ZINCRBY key increment member: 为有序集 key 的成员 member 的 score 值加上增量 increment
  • ZCARD key: 有序集 key 的长度
  • ZCOUNT key min max: 返回有序集 key 中 socre 在 min 到 max 之间的元素数量
  • ZREM key member: 删除元素
  • ZREMRANGEBYRANK key start stop: 移除排名在 start 到 stop 之间的元素
  • ZREMRANGEBYSCORE key min max: 移除 score 在 min 到 max 之间的元素
  • ZRANK key member: 返回 member 的排名
  • ZREVRANK key member: 返回 member 从大到小的排名

  • ZUNIONSTORE:

持久化

Redis 持久化提供两种方式: RDB 和 AOF(append-only-file)

RDB

rdb 文件保存的内容格式和使用 DUMP 命令后的数据格式相同

  • 根据配置的时间间隔自动保存快照
  • Redis客户端执行 SAVE, BGSAVE, FLUSHALL 命令是会保存快照
  • 执行复制(replication)时也会保存快照

RDB 相关配置

AOF

AOF(append-only-file) 按照特定的 Redis 协议格式将命令写入文件。

  • 默认每秒同步一次，所以使用这种方式做持久化，数据具备很高的安全性。
  • aof文件是只追加的日志文件，如果在写入时磁盘已满等语言造成的命令不完整，也可以使用 redis-check-aof 来修复
  • aof 文件过大时会自动在后台重写，重写后的文件小于等于现有的 aof 文件，并且整个过程是能保证一致性的。

经典模式学习

时间序列

APPEND 可以为一系列定长(fixed-size)数据(sample)提供一种紧凑的表示方式，通常称之为时间序列。

使用 bitmap 实现用户上线次数统计

限速器

假设现在我们希望记录自己网站上的用户的上线频率，比如说，计算用户 A 上线了多少天，用户 B 上线了多少天，诸如此类，以此作为数据，从而决定让哪些用户参加 beta 测试等活动.

安全队列

创建一个 List 列表，它不仅可以返回一个消息，同时还将这个消息添加到另一个备份列表当中，如果一切正常的话，当一个客户端完成某个消息的处理之后，可以用 LREM 命令将这个消息从备份表删除。这种数据结构就是安全队列。

循环列表

和构造安全队列的方式相似，但是在调用RPOPLPUSH时，使用两个相同的key，这样取出来的value又会加到队尾，形成循环列表

进阶学习篇

事务

事务是『将多个命令打包， 然后一次性、按顺序地执行』的机制，事务在执行期间不会中断。

ACID

  • 原子性(atomictiy):
    • Redis 中事务要么不执行，要么全部执行，即使期间某个命令执行失败
    • Redis 不提供回滚(rollback)机制 (好鸡肋。。)

  • 一致性(consistency):

    • RDB 模式下事务如果只成功执行了部分命令，将不会被写入到 rdb 文件中
    • AOF 做持久化时会将事务写入aof日志文件，但可能出现没有完整写到磁盘的情况，这个时候可以使用 redis-check-aof 移除部分事务命令来保证一致性

  • 隔离性(isolation):

    • Redis 服务是单进程的，在它执行事务时，会按序执行命令，并且不会被其他命令中断。
  • 持久性(durability):
    • 如果服务 cursh 了，很难保证事务的持久性。所以 Redis 的事务不具备持久性

Redis 事务基础命令

  • MULTI: 标记事务的开始
  • EXEC: 执行所有事务块的命令
  • DISCARD: 取消事务，放弃执行事务块内的所有命令。
  • WATCH key [key ...]: 监视 key，如果 key 在执行事务之前被修改，事务终端

事务应用场景

  • 银行应用

    1. 检查支票账户的月高于200美元
    2. 从支票账户余额减去200美元
    3. 在储蓄账户的月中增加200美元

  • Follow关系

    1. 用户A关注了用户B
    2. 向用户A的关注列表里 Push 用户B
    3. 向用户B的粉丝列表里 Push 用户A

Redis 中开始事务使用MULTI命令，执行事务使用EXEC命令。

Pub/Sub

Redis 提供消息订阅(Pub/Sub)模式来做事件通知(event notification)。

  • 内置 channel 模式

    • keyspace@db__:key，mssage 为 command
    • keyenvet@db__:command，message 为 key

  • 命令:

    • PUBLISH，发布模式，可发送多个频道，O(N+M)，N为频道数量，M为订阅者数量
    • PSUBSCRIBE，订阅模式，topic 模式，O(N)
    • SUBSRCIBE，订阅模式，direct 模式，O(N)
    • PUNSUBSCRIBE，取消订阅 topic 频道
    • UNSUBSCRIBE，取消订阅 dircet 频道
    • PUBSUB，自省，查看频道和订阅者

集群

文当地址: Redis集群教程

简介

Redis 集群是一个可以在多个 Redis 节点之间进行数据共享的设施。

  • 不支持需要同时处理多个键的 Redis 命令, 例如HMSET HMGET之类
    • 执行这些命令可能需要在多个 Redis 节点之间移动数据
    • 高负载的情况下， 这些命令将降低 Redis 集群的性能， 并导致不可预测的行为 (不可预测的行为是什么？)

集群的优点:

  • 将数据自动切分到多个节点的能力。（随机分布）
  • 当集群中的一部分节点失效或者无法进行通讯时， 仍然可以继续处理命令请求的能力。（利用分区主从关系）

集群数据共享

  • 每个节点各自负责一部分 hash slot，例如 A、B、C 分别负责序号为 0~5500、5501~11000、11001~16384 的槽
  • 新增一个节点时，可以指定一个或多个已存在的节点的部分槽转移至新节点
  • 移除节点时，该节点上的槽也可向其他节点转移
  • 主从复制：
    • 每个节点都可以有 1个主节点和 n 个从节点
    • 如果主节点挂掉，其从节点会有一个被选举成为主节点替代它继续工作
