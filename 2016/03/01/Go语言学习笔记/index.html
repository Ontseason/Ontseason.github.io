函数

函数在 Go 中是第一类对象。

不支持嵌套(nested)、重载(overload)和默认值参数(default parameter)

  • 无需声明原型
  • 支持多返回值
  • 支持命名返回参数
  • 支持匿名函数和闭包

函数签名

  • 位置连续的同一类型变量可以和合并

  • …Type表示变参，只能出现在最后一个位置，实际上是个 Slice，意味着可以是0个或多个值。

  • 如果是命名返回参数，则和函数参数一样存在栈中，隐式return会把栈中的返回值变量返回，意味着如果在函数里，对返回值变量没有任何赋值的操作的话，会返回对应类型的初始化值。

panic, recover 错误处理

输出

panic会中断函数的执行，进入 derfer 链的调用过程。可以想象成 Python 里的异常处理

区别在于 Python 捕获到异常之后可以选择继续 raise，让异常在执行堆栈中传递下去，或者是不传递异常而让函数继续执行。而在 Go 中，recover用于捕获异常，并且它只能在defer的直接调用中出现，不然就无法捕捉到异常了。

而之前说过，defer 链的执行是在函数return之前发生的，而一旦有panic出现，后面函数除了已注册的 defer 链和 return 之外会忽略其他任何操作。因此，一般在导致函数的执行流程无法继续下去的时候才使用panic，而可以容许执行流程继续时，用 error 来记录错误。

面向对象

如果类型 T 中包含匿名嵌入字段 M，则可通过 T 对象实例可直接访问到 M 的方法。

通过匿名字段，Worker 拥有了复用嵌入字段 Person 的成员和方法的能力。编译器如果在外层找到了方法的实现，则它会使用外层的定义方法，这又在某种程度上有了 override 的能力

使用方法的另一种显示调用形式: <type>.func(instance, args...)

选择器规则

Go 和 C 不同，对象和对象指针在引用成员和方法时，选择符都是使用.，所以针对这点，选择器有一套规则。

  • 类型 T 可引用全部 T 方法，但是 T 不能引用 T 的实现方法
  • 类型 T 包含匿名字段 M，T 可引用全部 M 能引用到的方法
  • 类型 T 包含匿名字段 M，T 可引用全部 M 能引用到的方法
  • 类型 T 包含字段 M 或 M，则 T 能引用全部 M 能引用到方法

总结，指针类型能引用到所指向内存区域的所有方法。普通对象实例不能引用指针类型实现的方法。

接口

关键词interface，接口的特点:

  • 接口命名习惯以 er 结尾
  • 接口没有数据成员
  • 可以在接口中嵌入其他接口
  • 只包含函数签名，没有实现

如果一个类型的方法集里包含了一个接口中的所有方法签名，那么这个类型就实现了这个接口。

假设有一个Speaker接口，目前只包含一个函数签名introduce()，而现有的Worker类型是实现了这个函数签名对应的方法的，所以Worker就实现了Speaker接口

如果接口 I 中还嵌入了接口 N，那么要实现 I 接口的类型必须也要实现 N 接口。

interface{} 没有任何方法签名，所以任何类型都默认实现了它。(TODO: 继续加深理解)

Channel

关键词chan <Type>，用于声明 channel 类型，内部实现是同步阻塞的，所以保证了并发安全。channel 是 Golang 在语言层面提供的消息机制，就是所谓的CSP(Communicating Sequential Processes)，这样 goroutine 之间就可以通过 channel，而不是一个全局对象进行通信了。

更赞的是，channel 还提供只读和只写这两种类型

  • chan<- <Type> 只写 channel
  • <-chan <Type> 只写 channel
