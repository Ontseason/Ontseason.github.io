对象模型概述

Python 中的对象(new class 诞生以后):

  • class对象: Python 中自定义类型
  • instance对象: 由class对象创建的实例

Python 中内置类型对象int, str本质都是class，在 Python2.2 之前它们是单独一套type类型，2.2之后类型机制重构后全部变成了class对象。

对象之间对应着两种关系:

  • is-kind-of 关系：基类与子类的关系，子类is kind of基类
  • is-instance-of 关系：类与实例的关系，实例is instance of类

从 type 对象到 class 对象

对象在 Python 中的定义

PyObject 定义中包含一个struct _typeobject*指针变量，它指向对象的类型，另外一个别名是 PyTypeObject。

<type 'object'>和<type 'type'>在 Python 中都是 PyTypeObject 结构体变量，分别对应 PyType_Type 和 PyBaseObject_Type 。

类的初始化

Python 启动后，初始化内置类型对象，而因为内置类型对象都是 PyTypeObject 变量，所以其实就是做了一些填充结构体中的重要字段的事。这个过程主要是在PyType_Ready中完成

Slot机制

Python 通过 slot 机制将类对象的初始方法以(name, &method)的形式添加到tp_dict当中，比如("__add__", &nb_add)。 Python定义的 slot 静态集合 slotdefs，集合了类对象中可能出现的所有magic method:

slot 机制会计算出 name 对应的函数指针在 PyHeapTypeObject 这个结构体中的偏移量 offset，当出现一个操作名可能对应多个操作函数时，比如 PyList_Type 中既有mp_subscript又有sq_item，这时候根据 PyHeapTypeObject 中域的顺序，就能算出 offset(mp_subscript) < offset(sq_item) 这样的结果了。

可是 soltdef 的结构体描述没有表现出callable 的性质，所以想让 slot 和 操作名关联起来，还需要将 slotdef 包装一下

将操作填充进 tp_dict 中是由add_operators来做的，对 slot 包装的过程就是在这里完成

Python3.3之后不再做排序操作，而是在 slotdefs 初始化之处就根据 offset 按序列好，在之前会在init_slotdefs中使用快排对 slotdefs 做排序操作的。另附题外话，思考。

metaclass

以创建一个用户自定义的类对象为例

class 元信息就是关于创建这个类对象所包含的全部信息，例如 class 名称，空间，拥有的属性和方法，实例化时申请的内存大小等。

当Python解释器在准备创建一个 class 对象之前，会先获取这个 class 的属性表(动态元信息)，class 名称和 class 基类列表，拿到这些属性以后打包成一个tuple，然后和这个类的 metaclass，在这里是type(根据MRO向上寻找，直到发现父类具有 metaclass 为止)，一起传入PyObject_Call函数中

PyType_Type的ob_type指向自己，它的tp_call指向了type_call函数，而type_call函数最终会调用type_new来创建这个类对象。

至此，一个用户自定义的类对象就诞生了。
