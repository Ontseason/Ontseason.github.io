闭包

先看一个闭包的例子

func_closure记录了闭包的位置。在编译时期就确定了free variables和cell variables，其实指的都是同一个对象，分别存储于不同函数的func_code.co_freevars和func_code.cell_freevars中

对于函数foo而言，嵌套函数bar使用了它的局部变量n，那n就是foo的cell variable，反过来说n就是bar的free variables，而闭包有一种说法就是函数和它的free variables构成。

gengrator

定义: 函数中包含yield声明。

gen.__next__()或gen.send(value)用于迭代，send可以传递值到= yield表达式左边的变量。初次启动generator如果使用send，参数必须是None。可以 if (yiled x), while (yield x), Py3.x 新增yiled from。

生成器函数被调用时返回一个生成器。使用生成器的__next__或send方法，恢复生成器函数内部的执行，直到在函数内部再次遇到yield，此时会保存当前执行上下文，堆栈指针，暂停函数执行并把yield持有的值返回，将程序执行控制权交给外部调用部分的代码。

Py3.3 之前不允许生成器函数中的return带返回值，Py3.3 之后可以有，用作中止异常的参数。

gen.close()关闭生成器，如果在函数内部yield部分有做try..finally捕捉，最终会执行finally部分的代码。

gen.throw(exce)，传递异常到生成器函数内部，生成器中可以捕获处理。

比较好的说明throw好处的回答: what is generator good for?

metaclass

metaclass 用于生成类对象，它可以是任何可调用的对象，无论是类，对象，函数，只要能够根据类的动态元信息生成类对象即可。使用 metaclass 时传递类名、基类tuple、包含类属性的 dict 这三个参数

观察生成类对象的过程，将 __metacalss__ 换成类，以Python3.x为例

metaclass的__new__方法创建了想要的类对象，调用类对象，进入到__call__生成对象实例，所以__call__必须包含生成和初始化实例的两个方法: __new__和__init__，执行完后返回实例。

decorator

装饰器是函数嵌套的语法糖，如果把它的调用层展开，就很容易理解

descriptor

__getattr__、__getattribute__、__get__的关系:

  • __getattribute__(self, name): 获取实例属性的默认方法
  • __getattr__(self, name): 属性在对象和类中都没有找到，__getattribute__就会调用getattr方法尝试获取属性。
  • __get__(self, instance, owner): 如果属性中含有__get__方法就会调用

描述符可以含有__get__、__set__、__delete__中的一个或多个方法。

作用:

  • __get__: 对让外部访问的属性做统一的预先处理。
  • __set__: 设置属性时可做预先处理或判断，如果描述符中不含__set__方法，则属性变成只读
  • __delete__: 删除属性时绑定相关操作
