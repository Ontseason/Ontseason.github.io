Tornado源码分析系列之一: gen

发表于 2016-02-22

Tornado 几乎是 Python 关于异步必会提及的框架，在公司负责的业务虽然很少用到，不过 Tornado 以精巧简洁的设计和十分优异的性能著称，对它是如何实现的也十分好奇和感兴趣，所以在阅读了一些源码之后也试着整理思路简单分析一下。

这段代码的神奇之处在于被gen.coroutine修饰的函数内部, 凡是需要异步调用的过程只需加上 yield，就能让获取异步执行的结果看起来和”赋值”一样简单。

这种实现带来了非常赞的效果:

  • 代码流程和同步写法近似（多个 yield
  • 不用在各种回调函数里跳来跳去
  • 可以yield list 和 dict，来达到类似map()的效果

这是精简之后的主要流程代码，result 是被装饰的函数返回的生成器, future 是 TracebackTrue 类对象，顾名思义用来记录执行过程中可能出现的异常, next(result)启动生成器，并返回一个Future，如果生成器迭代结束，设置函数返回值，不然就进入到Runner中。

来分析handled_yiled的执行过程。回到前面给出的实例代码 response = yield http_client.fetch(url)，在执行到这句时，http_client.fetch(url) 其实会返回一个 Future 对象，所以在handled_yield中，保存这个值的变量yielded经过条件判断，会被io_loop加入到外层的事件循环中等待处理，并将self.run注册为它的回调函数，然后返回 False，等到Future有结果时，就会调用相应的回调函数去处理它。而如果此时Future已经有结果准备好了，会返回 True 并立刻执行self.run。所以接下来再来看看这个 self.run 又做了哪些工作

根据While True部分的代码，很容易看出self.run 其实就是不断从有结果的Future对象中将结果取出，发送到生成器内部，再继续执行handled_yield，这个过程会一直持续下去，直到生成器迭代结束，最终设置函数返回值。
