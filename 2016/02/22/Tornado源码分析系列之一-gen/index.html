Tornado源码分析(一): gen

发表于 2016-02-22

引言

Tornado 几乎是 Python 关于异步必会提及的框架，在公司负责的业务虽然很少用到，不过 Tornado 以精巧简洁的设计和十分优异的性能著称，对它是如何实现的也感到很好奇，在自己阅读几遍源码之后也试着整理思路简单分析一下。

先从一个简单的例子开始:

这段代码的神奇之处在于：被gen.coroutine装饰的函数内部, 凡是需要异步调用的过程只需加上 yield，就能让获取异步执行的结果看起来和”赋值”一样简单。

这样的实现方式带来了一些非常赞的效果:

  • 异步代码流程和同步写法近似（多了个 yield
  • 不用在各种回调函数里跳来跳去
  • 可以yield list 和 dict，来达到类似map()的效果

Future

来一探究竟gen.coroutine到底做了哪些神奇的事情，首先来看异步实现中一个重要概念Future:

既然是异步调用，那同步代码肯定没办法知道结果什么时候会被执行出来，所以Future做的就是存储结果或者执行异常，记录执行状态。

Cortinue

再来看cortinue装饰器的实现:

这是精简之后的主要流程代码，result 是生成器，future 是 TracebackFuture 实例对象，用来记录执行过程中的错误和traceback，需要注意的是，因为在 Python3.3 之前，生成器函数中的 return 是不能带任何返回值的，所以为了返回函数结果，Tornado里使用了raise gen.Return(result)这种方式来替代return result，而函数结果最后也会被以异常处理的方式捕捉到并且记录在 future 里。

然后next(result)启动生成器，开始初次迭代并返回一个Future。这里其实还有一个比较有趣的stack_context的机制，正是因为它，才让callback回调函数无论在哪个context被执行，最终都可以被用户代码异常处理，具体机制之后再聊。接下来代码会进入到Runner中。
